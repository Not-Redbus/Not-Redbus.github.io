01. Problem Statement

Downstream logistics systems were crashing because IoT devices sent null values during connectivity dips. We needed a gateway middleware to reject malformed payloads before they hit the database.

02. Acceptance Criteria (Gherkin)
GIVEN an incoming Shipment_Event payload
WHEN the field "current_temp" is missing OR null
THEN reject request with 400 Bad Request
AND log the "device_id" for maintenance review.
03. Technical Constraints
  • Latency must remain under 50ms per validation.
  • Must support JSON Schema Draft-07.
